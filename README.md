# Bank Term Deposit Prediction (Mid-term Project)

## 📌 Опис задачі
Мета проєкту — побудувати модель машинного навчання, яка передбачає, чи відкриє клієнт банку строковий депозит (бінарна класифікація), спираючись на демографічні дані, історію контактів та макроекономічні показники.

## 🛠 Що було зроблено
1. **EDA:** Проведено розвідувальний аналіз, виявлено сильний дисбаланс класів (~11% позитивного класу).
2. **Препроцесинг:** Видалено ознаку `duration` (щоб уникнути витоку даних), згруповано категорії освіти, оброблено викиди (clipping для `campaign`) та створено нові логічні ознаки (наприклад, `was_contacted_before`).
3. **Моделювання:** Натреновано 4 базові моделі: Logistic Regression, kNN, Decision Tree та XGBoost.
4. **Тюнінг:** Для XGBoost проведено оптимізацію гіперпараметрів за допомогою `RandomizedSearchCV` та `Hyperopt`.
5. **Інтерпретація:** Проведено аналіз важливості ознак та впливу на передбачення за допомогою `SHAP`.

## 💡 Висновки та шляхи поліпшення
Найкращою моделлю виявився **XGBoost**, налаштований через Hyperopt. Модель показала стабільний результат без перенавчання.
Згідно з SHAP-аналізом, найважливішими факторами для клієнтів є макроекономічні показники (рівень зайнятості, ставки) та факт попереднього успішного контакту.

**Як можна поліпшити результати:**
* Оптимізувати поріг класифікації (Threshold) з 0.5 до нижчого значення, оскільки більшість False Negatives мають ймовірність близько 0.3-0.4.
* Застосувати Cost-sensitive learning (надати різну вартість помилкам FP та FN).
* Спробувати інші алгоритми (CatBoost, LightGBM) або їх ансамблювання (Stacking).
